# Natural Hub — Backend Architecture & Requirements

هذا المستند يوضح التفاصيل التقنية اللازمة لبناء النظام الخلفي (Backend) لمركز Natural Hub، مع مراعاة تبسيط تجربة العميل (بدون حسابات) وقوة الإدارة (3 فروع، تقارير، وتوزيع موظفين).

---

## 1. رؤية النظام (System Vision)
- **للعميلة:** تجربة حجز سريعة جداً — فقط اختيار الخدمة، الفرع، وترك رقم الموبايل.
- **للإدارة:** نظام مركزي لإدارة 3 فروع، متابعة الإيرادات، وتكليف الموظفات بالمهام.

---

## 2. هيكلية قاعدة البيانات (Database Schema)

### الجداول الأساسية:

| الجدول | الأعمدة الرئيسية |
|---|---|
| `branches` | `id`, `name`, `address`, `phone`, `status` (active/inactive) |
| `services` | `id`, `name`, `category` (Hair/Skin/Lashes/Shop), `price`, `duration`, `description`, `icon`, `branch_id` |
| `staff` | `id`, `name`, `branch_id`, `specialty`, `phone`, `status` |
| `customers` | `id`, `phone` ***(Unique — بديل الأكاونت)***, `name`, `total_visits`, `last_visit` |
| `bookings` | `id`, `customer_id`, `service_id`, `branch_id`, `staff_id` (nullable), `appointment_date`, `status` (Pending/Confirmed/Completed/Cancelled) |
| `payments` | `id`, `booking_id`, `amount`, `method` (Cash/Card/Wallet), `created_at` |

### جداول المحتوى (CMS Tables):

| الجدول | الأعمدة الرئيسية |
|---|---|
| `gallery_images` | `id`, `url`, `category` (Hair/Skin/Lashes), `sort_order`, `created_at` |
| `videos` | `id`, `title`, `file_url`, `poster_url`, `category`, `sort_order` |
| `blog_posts` | `id`, `title`, `body` (HTML), `thumbnail_url`, `category`, `published_at` |
| `shop_products` | `id`, `name`, `description`, `price`, `image_url`, `bundle_id` (nullable), `in_stock` (boolean) |

## 3. منطق الحجز (Booking Logic — بدون حسابات)

### قناتين للحجز (Dual Channel):

**القناة 1 — حجز ذاتي من الموقع (Online Booking):**
```
العميلة تفتح الموقع → تختار الفرع → تختار الخدمة → تدخل رقمها واسمها
        ↓
النظام يبحث في جدول customers
        ↓
 ┌─────────────────────┬──────────────────────────┐
 │ رقم موجود مسبقاً    │ رقم جديد                 │
 │ → يربط الحجز بملفها │ → ينشئ ملف عميلة جديد   │
 └─────────────────────┴──────────────────────────┘
        ↓
يُنشئ سجل حجز بحالة "Pending"
        ↓
يُرسل تأكيد عبر WhatsApp API تلقائياً
        ↓
يظهر في لوحة التحكم → المدير يعيّن موظفة عليه
```

**القناة 2 — حجز يدوي من الأدمن (Manual Entry):**
```
عميلة تتصل بالتليفون أو تيجي Walk-in
        ↓
الموظفة تفتح لوحة التحكم → "إضافة حجز جديد"
        ↓
تدخل رقم العميلة → النظام يعرض بيانات العميلة لو موجودة
        ↓
تختار الفرع + الخدمة + التاريخ + الموظفة المسؤولة
        ↓
يُنشئ سجل حجز بحالة "Confirmed" مباشرة
```

### الفرق بين القناتين:

| | حجز من الموقع | حجز يدوي من الأدمن |
|---|---|---|
| **مين بيحجز؟** | العميلة بنفسها | الموظفة أو المدير |
| **الحالة المبدئية** | Pending (تحتاج تأكيد) | Confirmed (مؤكد فوراً) |
| **تعيين الموظفة** | لاحقاً من لوحة التحكم | أثناء إنشاء الحجز |
| **مصدره** | `source: website` | `source: manual` |

### لماذا القناتين مهمين؟
- **الموقع** يقلل الضغط على التليفونات ويسمح للعميلات بالحجز 24/7.
- **اليدوي** ضروري للعميلات اللي بتتصل أو بتيجي مباشرة بدون ما تفتح الموقع.
- **التقارير** تقدر تفرق بين مصادر الحجوزات لمعرفة "هل الموقع بيجيب حجوزات فعلاً؟"

### مميزات نظام الحجز:
- صفر احتكاك للعميلة (لا إيميل، لا باسورد، لا تفعيل).
- بناء تلقائي لقاعدة عملاء دون أي جهد إضافي.
- يمكن مستقبلاً تفعيل برنامج "نقاط ولاء" بنفس الرقم.

## 4. نقاط الاتصال بالـ Frontend الحالي (API Endpoints)

| الـ Endpoint | الطريقة | الوظيفة |
|---|---|---|
| `/api/book` | POST | إرسال بيانات الحجز من الموقع |
| `/api/services` | GET | جلب الخدمات والأسعار ديناميكياً |
| `/api/branches` | GET | جلب قائمة الفروع |
| `/api/gallery` | GET | جلب صور المعرض مصنفةً حسب القسم |
| `/api/videos` | GET | جلب الفيديوهات مصنفةً حسب القسم |
| `/api/blog` | GET | جلب مقالات المدونة مع إمكانية الفلترة |
| `/api/shop/products` | GET | جلب منتجات المتجر |
| `/api/admin/bookings` | GET | جلب كل الحجوزات (للإدارة فقط) |
| `/api/admin/staff/:id/assign` | PUT | تعيين موظفة على حجز معين |

## 5. نظام إدارة المحتوى (Dynamic CMS)

لضمان أن يكون الموقع حياً ومتجدداً بدون الحاجة للمبرمج:

### أ. إدارة الخدمات (Services)
- إضافة/حذف أقسام كاملة (مثل إضافة قسم "المساج" مستقبلاً).
- تعديل أسعار الجلسات، أيقوناتها، وأوصافها — يظهر التحديث فوراً في الموقع.
- تحديد إذا كانت الخدمة متاحة في فرع معين أم لا.

### ب. معرض الصور (Gallery)
- واجهة رفع صور سهلة مع اختيار القسم (شعر، بشرة، رموش).
- ضبط الترتيب عبر Drag & Drop.
- حذف أو إخفاء الصور دون حذفها نهائياً.

### ج. الفيديوهات (Videos)
- يدعم رفع ملفات الفيديو مباشرة (MP4/MOV) أو إضافة روابط خارجية.
- تحديد "صورة الغلاف" (Poster) لكل فيديو.
- تصنيف الفيديوهات حسب (شعر، بشرة، رموش).

### د. المدونة (Blog)
- محرر نصوص غني (Rich Text Editor) لكتابة المقالات.
- رفع الصور المميزة (Thumbnail) لكل مقال.
- جدولة نشر المقالات (Scheduled Publishing).

### هـ. المتجر (Shop)
- إضافة/تعديل منتجات "الأفتر كير" مع الصور والأسعار.
- إنشاء "مجموعات" (Bundles) ووضع سعر خاص للباقة.
- تحديد حالة المخزون (In Stock / Out of Stock).

## 6. نظام الإدارة والتقارير (Admin & Analytics)

### تقارير الفروع:
- **إيرادات كل فرع:** فلترة يومية/شهرية/سنوية.
- **نسبة الإشغال:** عدد الحجوزات المكتملة مقارنة بالطاقة الكاملة.
- **مقارنة الفروع:** رسم بياني يوضح أداء الفروع الثلاثة جنباً إلى جنب.

### تقارير الخدمات:
- **أكثر الخدمات طلباً** لمعرفة أي الأقسام تحتاج تركيزاً أو عروضاً.
- **الخدمات الأكثر ربحية** (الإيراد الكلي من كل خدمة).

### إدارة الموظفات:
- لوحة تحكم للمدير يرى فيها الحجوزات "المعلقة" ويعين الموظفة المناسبة.
- تقرير أداء الموظفة (كم حجز أكملت؟ كم إيراد جلبت؟).

## 8. الصلاحيات (Roles & Permissions)

| الدور | ما يستطيع رؤيته وتعديله |
|---|---|
| `Super Admin` | كل شيء — كل الفروع، كل التقارير المالية، إدارة المحتوى |

## 9. خارطة التنفيذ (Phased Roadmap)

```
المرحلة الأولى (1-2 أسبوع)
├── بناء قاعدة البيانات
├── API الحجز الأساسي
└── ربط فورمة الحجز في الموقع

المرحلة الثانية (2-3 أسبوع)
├── لوحة تحكم الفروع
├── نظام إدارة الموظفات
└── CMS (الخدمات، المعرض، الفيديو)

المرحلة الثالثة (أسبوع)
├── التقارير المالية المتقدمة
├── CMS (المتجر، المدونة)
└── ربط WhatsApp API للتأكيدات
```

## 10. التقنيات المقترحة (Tech Stack)

| الطبقة | الخيار المقترح | السبب |
|---|---|---|
| **Backend** | PHP (Laravel) | سهولة على مطورين مصريين، مجتمع كبير |
| **Database** | MySQL | علاقات واضحة وسرعة في التقارير |
| **Storage** | Local / S3 | لحفظ صور وفيديوهات المعرض |
| **Auth** | Laravel Sanctum | لحماية الـ Admin APIs |
| **Notifications** | WhatsApp Business API | تأكيد المواعيد تلقائياً |
| **Hosting** | Linux VPS | سرعة وتحكم كامل |
| **SSL** | Let's Encrypt | تشفير مجاني |
| **Docs** | Postman Collection | توثيق كل الـ APIs للفريق |

## ملاحظة ختامية

> النظام ليس مجرد "فورمة حجز"، بل هو **ERP مصغر** يربط الموقع الإلكتروني بالإدارة المالية والتشغيلية للفروع الثلاثة.
>
> مصمم ليكون **قابل للتوسع (Scalable)** — إضافة الفرع الرابع تتم بضغطة زر من لوحة التحكم، بدون أي تعديل في الكود.
